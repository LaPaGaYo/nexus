# Nexus Design Governance Reference

Use this reference when a Nexus skill says to lazy-load design governance.

## Core Asset Protocol

For named brands or products, ask for or locate real assets before inventing visuals:

- logo
- product renders or photography for physical products
- UI screenshots for digital products
- color palette
- typography
- brand guidelines

If assets are missing, search official channels before improvising:

- `<brand>.com/brand`
- `<brand>.com/press`
- `brand.<brand>.com`
- product pages, launch films, app-store screenshots

Prefer real logo, product, and UI assets over inferred colors or "brand vibes". Verify logo fidelity, image resolution, UI freshness, and extracted colors. Freeze the result to `brand-spec.md` so later design work consumes the same truth instead of re-guessing.

Asset importance order:

1. Logo
2. Product renders for physical products
3. UI screenshots for digital products
4. Color values
5. Fonts

If brand assets cannot be verified, say so explicitly. Do not fake brand accuracy from memory.

## Direction Fallback

If the brief is visually vague or the design context is thin, propose 3 differentiated directions. They must differ in composition, typography, palette, and tone. Explain why each direction fits the product and what tradeoff it makes.

Do not present "minimal / bold / premium" as distinct directions. The user should feel the contrast immediately.

## Direction Architecture

When generating multiple directions:

- default to 3 directions
- anchor each direction to a named designer or studio
- use different schools, such as editorial systems, motion poetics, luxury restraint, avant-garde, or contemplative/eastern design
- state the direction name, anchor, fit, visual traits, and risk

## Junior-Designer Discipline

Do not rely on heroic one-shot guesses. Start with assumptions, show something concrete quickly, then tighten with real content, assets, and constraints.

## Five Design Lenses

Use these lenses when proposing, reviewing, or finalizing design work:

1. Philosophical coherence
2. Visual hierarchy
3. Execution craft
4. Functional fit
5. Distinctiveness

When critiquing, bias toward Keep, Fix, and Quick Wins.
