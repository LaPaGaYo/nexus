# Nexus v1.0.2 Release Prep Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `v1.0.2` as the next Nexus release so the maintainer automation loop and refresh-contract fixes become the new upgradeable stable baseline.

**Architecture:** Keep Nexus as the sole release contract owner. This milestone does not add new runtime subsystems; it advances the published release contract by updating the version markers, writing canonical release notes, running the Nexus-owned maintainer gate, then publishing and smoke-verifying the tagged release. All evidence remains repo-visible through release notes and a closeout artifact.

**Tech Stack:** Bun, Bash release helpers in `bin/`, TypeScript contracts under `lib/nexus/`, Git tags, GitHub Releases, repo-visible docs under `docs/releases/` and `docs/superpowers/`.

---

  - Move the published release contract from `v1.0.1` to `v1.0.2`.
- Create: `docs/releases/2026-04-10-nexus-v1.0.2.md`

- Create: `docs/superpowers/closeouts/2026-04-10-nexus-v1.0.2-release-closeout.md`
  - Capture the maintainer gate, publication evidence, remote smoke result, and final release state.
