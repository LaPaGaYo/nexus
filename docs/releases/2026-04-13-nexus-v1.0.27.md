# Nexus v1.0.27

Date: 2026-04-13
Tag: `v1.0.27`
Status: `release_candidate`

## Summary

This patch release closes the remaining contract-drift bug in the fresh-run
worktree model. Nexus now mirrors the active run contract into the run-owned
worktree before governed execution, so Codex and Gemini read the same product
and planning artifacts that Claude just wrote at the repo root.

## What Ships

- run-owned worktrees now receive mirrored `.planning/current`,
  `.planning/nexus/current-run.json`, and `docs/product` artifacts after
  canonical writeback for `discover`, `frame`, and `plan`
- governed `/handoff`, `/build`, `/review`, `/qa`, and `/ship` now resync the
  run contract into the execution worktree before provider dispatch, preventing
  stale phase packets from reopening older scope
- closeout cleanup now ignores mirrored Nexus-owned run artifacts when deciding
  whether a retired worktree is safe to remove, while still retaining worktrees
  that contain real code changes

## Notes

- This is a patch release on top of `v1.0.26`.
- The fix keeps the control plane split introduced in `v1.0.26`:
  - repo root remains the session/control root for CCB
  - execution still happens inside the run-owned worktree
  - only run contract artifacts are mirrored across that boundary
