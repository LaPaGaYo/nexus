# Nexus v1.0.29

Date: 2026-04-14
Tag: `v1.0.29`
Status: `release_candidate`

## Summary

This patch release fixes governed CCB dispatches that printed a valid
canonical build, review, or QA payload but still exited non-zero. Nexus now
recovers those provider results instead of falsely marking the stage blocked.

## What Ships

- governed CCB `build`, `review`, and `qa` dispatches now attempt stage-aware
  payload recovery from `stdout`/`stderr` before concluding that a non-zero
  `ask` exit means the provider blocked
- blocked CCB raw output now preserves `exit_code`, `stdout`, `stderr`, and
  `request_id`, so late or noisy provider failures stay debuggable from
  repo-visible artifacts
- governed review no longer loses a valid Gemini or Codex audit when the
  provider emitted the canonical payload but the surrounding shell command
  returned a non-zero exit

  semantics beyond correctly accepting already-emitted canonical payloads.
