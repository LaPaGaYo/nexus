# Nexus v1.0.36

Date: 2026-04-20
Tag: `v1.0.36`
Status: `release_candidate`

## Summary

This patch release tightens governed review completion and the user-facing
handoff between the tail lifecycle stages. Nexus now reads the canonical final
audit block from governed review providers, requires an explicit disposition
for passed advisories before the run can continue, and surfaces clearer
interactive next-step guidance from `/build` through `/closeout`.

## What Ships

- governed CCB review now extracts the final canonical `# Codex Audit` /
  `# Gemini Audit` block instead of trusting earlier stray `Result:` lines, so
  the release gate follows the true final audit verdict
- passed `/review` runs that include advisories now persist canonical advisory
  artifacts and require an explicit disposition:
  `fix_before_qa`, `continue_to_qa`, or `defer_to_follow_on`
- advisory-driven fix work now routes canonically through
  `/build -> /review -> /qa` without reclassifying advisories as hidden review
  failures
- `/qa`, `/ship`, and `/closeout` now block until review advisories are
  dispositioned, preventing silent carry-through of unresolved advisory debt
- the generated stage skills for `/build`, `/review`, `/qa`, `/ship`, and
  `/closeout` now provide explicit completion interactions and side-skill entry
  points instead of leaving all next-step routing implicit
