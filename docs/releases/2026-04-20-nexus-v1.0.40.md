# Nexus v1.0.40

Date: 2026-04-20
Tag: `v1.0.40`
Status: `release_candidate`

## Summary

This patch release hardens governed review and QA transport recovery so late or
partial CCB results stop polluting current truth and false-start foreground
dispatches recover automatically.

## What Ships

- `/review` now persists review-attempt identity and bound audit request ids so
  retry attempts stay isolated from stale late provider replies
- `nexus-ledger-doctor` now detects split-brain current audit sets instead of
  reporting a false clean state when `codex.md`, `synthesis.md`,
  `gate-decision.md`, `meta.json`, and `review/status.json` disagree
- CCB foreground false-start recovery is now provider-agnostic across Codex and
  Gemini for governed review and QA dispatches
- governed dispatch state now records `superseded` when a newer dispatch
  replaces an active one, instead of collapsing that state into `blocked`
- dispatch records now include retry provenance for false-start recovery,
  including initial exit code, missing request id, and stdout/stderr excerpts for
  postmortem diagnosis

## Notes

- This release is reliability-focused. It closes a real governed-execution gap:
  failed review attempts and false-started provider dispatches no longer need
  manual `.planning` surgery to recover cleanly.
- The dispatch-state additions are intentional operator tooling, not incidental
  logging. They make CCB transport failures auditable at the repo level.
