# Nexus v1.0.34

Date: 2026-04-19
Tag: `v1.0.34`
Status: `release_candidate`

## Summary

This patch release closes the remaining governance gaps around QA re-entry and
post-ship landing, then finishes the active product-surface rename to Nexus.

## What Ships

- `/qa` now distinguishes blocking findings from non-blocking advisories, and a
  failing QA gate can route into the canonical `qa -> build -> review -> qa`
  fix-cycle instead of forcing manual ledger edits or fake review failures
- `/ship` now persists SHA-bound PR handoff data, while `/land-and-deploy`
  records explicit `deploy-result.json` outcomes for pre-merge CI failures,
  merge conflicts, merge-queue failures, and post-merge deploy failures,
  including the canonical re-entry action
- deploy setup and landing now classify deploy surfaces generically as one
  primary surface plus optional secondary surfaces, so auxiliary/background
  deploy paths only warn instead of blocking the primary land path
- live Nexus surfaces no longer emit legacy `GStack` or `Garry Tan` branding:
  generated skills, design CLI helpers, CI/evals paths, and active docs/test
  fixtures are all Nexus-owned
