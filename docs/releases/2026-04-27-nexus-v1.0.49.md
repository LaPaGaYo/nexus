# Nexus v1.0.49

Date: 2026-04-27
Tag: `v1.0.49`
Status: `release_candidate`

## Summary

This release makes Claude local-provider mode more capable and fixes the
runtime regressions found while dogfooding a single-provider Nexus run.

## What Ships

- Claude local-provider now supports an explicit `agent_team` topology for
  Claude Code Agent Teams.
- `/frame`, `/plan`, `/build`, `/review`, `/investigate`, and `/ship` now
  surface a stage-aware local topology chooser for Claude hosts.
- `/handoff` validates Claude Agent Teams with a version and feature-flag check
  before allowing the route.
- Review audit artifacts are mirrored into run-owned worktrees, so
  `.planning/audits/current/*` is visible from the execution workspace.
- Review advisory extraction now counts inline `ADVISORY` bullets inside audit
  findings, not only a separate `Advisories:` section.
- QA now treats malformed optional `learning_candidates` as discardable
  ancillary data instead of blocking an otherwise valid QA PASS result.

- This is the upgrade users should take before retrying a local-provider QA run
  that was blocked only by malformed optional `learning_candidates`.
- Claude Agent Teams remain opt-in and require Claude Code support plus
  `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`.
