# Nexus v1.0.45

Date: 2026-04-26
Tag: `v1.0.45`
Status: `release_candidate`

## Summary

This patch release tightens the `/nexus` session bootstrap when a host has a
saved `governed_ccb` preference but the current session cannot run the governed
route. Nexus now stops before lifecycle work and asks whether to switch the host
to a runnable local provider or keep governed CCB and mount the missing
providers.

## What Ships

- `/nexus` now treats blocked saved `governed_ccb` as a route-choice moment
  instead of only printing a warning
- Claude hosts use `AskUserQuestion` to choose between switching to
  `local_provider` and keeping `governed_ccb`
- Codex/agents hosts use a plain numbered choice and wait for the user's answer
  before continuing
- choosing local provider persists `execution_mode`, `primary_provider`, and
  `provider_topology` from the detected local fallback
- root `/nexus` guidance now explicitly prevents lifecycle continuation until
  the blocked route choice is resolved

- The local-provider choice is persisted intentionally. Users who want to return
  to governed CCB can set `execution_mode governed_ccb` after mounting the
  required providers.
