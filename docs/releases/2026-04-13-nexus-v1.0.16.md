# Nexus v1.0.16

Date: 2026-04-13
Tag: `v1.0.16`
Status: `release_candidate`

## Summary

This patch release formalizes bounded fix-cycle review scope. When governed
review fails, Nexus now records the blocking findings as a repo-visible
`review_scope`, carries that scope through handoff/build/review, and instructs
Codex/Gemini to treat out-of-scope concerns as advisories instead of reopening
the whole phase as a fresh blocking review.

## What Ships

- failing governed `/review` now derives a structured `review_scope` with:
  - `mode`
  - `source_stage`
  - `blocking_items`
  - `advisory_policy`
- governed handoff refresh and fix-cycle `/build` now persist that
  `review_scope` in canonical status and request artifacts, so the fix cycle is
  defined by repo-visible contract rather than Claude synthesis alone
- governed and local review prompts now distinguish:
  - full acceptance review
  - bounded fix-cycle review
- bounded fix-cycle review prompts now instruct auditors to:
  - decide `pass|fail` only against the listed blocking items
  - record extra concerns as advisories only
  - avoid turning unrelated phase gaps into new blocking failures
- governed build result and governed handoff markdown now surface the active
  review scope, so the bounded contract is legible from repo artifacts
