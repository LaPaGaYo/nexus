# Nexus v1.0.8

Date: 2026-04-11
Tag: `v1.0.8`
Status: `release_candidate`

## Summary

This patch release focuses on clearer post-upgrade behavior in Claude, so
upgraded installs do not blur repo mode and execution mode.

## What Ships

- post-upgrade guidance that prints the effective execution path alongside repo
  ownership state instead of leaving `solo` / `collaborative` ambiguous
- a Claude-side upgrade prompt that persists `governed_ccb` vs
  `local_provider` when the machine has CCB but no saved execution-mode
  preference yet
- docs and generated skill surfaces aligned with the new upgrade-time
  execution-mode explanation
