# Nexus v1.0.20

Date: 2026-04-13
Tag: `v1.0.20`
Status: `release_candidate`

## Summary

This patch release closes the next real lifecycle gap exposed immediately after
`/closeout`. Nexus previously completed the work unit correctly, but then
forced users to hand-delete `.planning/nexus/current-run.json` before a new
`/discover` could start.

## What Ships

- `/discover` now recognizes a completed `closeout` as a finished run and
  automatically rolls into a fresh run id
- the previous run ledger is archived to
  `.planning/archive/runs/<run_id>/current-run.json`
- the previous closeout snapshot is archived alongside it under
  `.planning/archive/runs/<run_id>/closeout/`
- users no longer need to manually delete `current-run.json` to begin the next
  phase after a valid closeout

## Notes

- This is a patch release on top of `v1.0.19`.
- The new rollover path is intentionally narrow: it only triggers when the
  existing run is already in a completed `closeout` state with a matching ready
  closeout status artifact.
