# Nexus v1.0.13

Date: 2026-04-12
Tag: `v1.0.13`
Status: `release_candidate`

## Summary

This patch release tightens governed build/review execution so failing dual
review now enters a canonical fix cycle and build completions can no longer
self-certify from prior build-stage artifacts.

## What Ships

- `/build` now rejects generator summaries that try to prove completion from
  existing `.planning/current/build/*` or `.planning/nexus/current-run.json`
  instead of current implementation state
- generator summaries that explicitly return `- Status: blocked` now keep the
  build stage blocked instead of being normalized into a completed build
- `/review` now records a real `gate_decision` of `pass` or `fail` from the
  Codex/Gemini dual audits instead of hard-coding `pass`
- a failing governed review now allows the canonical transition `review -> build`
  for a bounded fix cycle, without rewriting handoff artifacts or backtracking
  to `/handoff`
- QA, ship, and closeout now require a passing review gate, while closeout
  history accepts governed fix-cycle loops of `build -> review -> build -> review`

## Notes

- This is a patch release on top of `v1.0.12`.
- The user-visible change is that a failing dual review now sends the work unit
  back to `/build` as a governed fix cycle, rather than forcing a manual
  handoff rewrite or a local override.
