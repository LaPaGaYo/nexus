# Nexus v1.0.24

Date: 2026-04-13
Tag: `v1.0.24`
Status: `release_candidate`

## Summary

This patch release fixes a governed CCB execution bug exposed in real
`/handoff` runs. Nexus was launching CCB commands with the correct process
`cwd`, but the child environment still carried the install directory in `PWD`.
CCB tools that resolve project scope from `PWD` could therefore miss mounted
providers and session files even when the repo-scoped sessions were healthy.

## What Ships

- CCB runtime commands now set `PWD` to the execution workspace for
  `ccb-mounted`, `ccb-ping`, `ask`, and `autonew`
- local-provider runtime commands also align `PWD` with the execution workspace
  for consistency
- governed handoff no longer misreads project-scoped mounted providers because
  a child process inherited the Nexus install directory in `PWD`

  processes got the right `cwd` but the wrong `PWD`.
