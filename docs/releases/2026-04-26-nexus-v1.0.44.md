# Nexus v1.0.44

Date: 2026-04-26
Tag: `v1.0.44`
Status: `release_candidate`

## Summary

This patch release fixes the completion-advisor interaction layer for
design-aware and externally installed skill recommendations. Persisted advisor
writes now keep the verification matrix and project cwd context, `/build`
promotes design/browser follow-ups into an interactive recommendation path, and
CLI hosts can print a host-independent chooser with `--output interactive`.

## What Ships

- persisted completion advisors now rank external installed skills with the
  same verification-matrix context used by the stage runtime
- `/build` now switches to `recommended_choice` when `/design-review` or
  `/browse` follow-ups are present, keeping post-build UI/UX checkpoints visible
- stage commands now pass project cwd and verification context into every
  completion-advisor write path
- `--output interactive` now renders the runtime-owned advisor choices in
  terminal hosts that cannot display `AskUserQuestion`
- external installed skill recommendations are deduplicated by command surface
  across install roots

## Notes

- The interactive CLI renderer does not auto-execute the selected command. It is
  a host-independent fallback for shells and clients that cannot show a native
  choice popup.
