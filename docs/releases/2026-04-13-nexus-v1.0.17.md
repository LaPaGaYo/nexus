# Nexus v1.0.17

Date: 2026-04-13
Tag: `v1.0.17`
Status: `release_candidate`

## Summary

This patch release closes the last bounded fix-cycle review gap behind stale
`v1.0.16` build artifacts. When a governed fix-cycle build predates the
persisted `review_scope` field, Nexus now recovers the bounded review scope
from the prior failing review instead of silently falling back to a full phase
acceptance audit.

## What Ships

- governed `/review` now recovers bounded fix-cycle scope from the previous
  failing review when the current fix-cycle build artifact omits
  `review_scope`
- governed review trace artifacts now persist `review_scope` directly in
  `.planning/current/review/adapter-request.json`
- Codex and Gemini now receive the same repo-visible bounded scope even when
  the fix-cycle build was produced by an older Nexus version
- stale fix-cycle builds no longer cause split review behavior where one
  auditor reruns full acceptance while the other audits only the intended
  blocking items

## Notes

- This is a patch release on top of `v1.0.16`.
- This change specifically repairs governed fix-cycle review for repositories
  whose `.planning/current/build/status.json` predates persisted
  `review_scope` but whose prior failing review already defined the bounded fix
  contract.
