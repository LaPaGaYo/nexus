# Nexus v1.0.26

Date: 2026-04-13
Tag: `v1.0.26`
Status: `release_candidate`

## Summary

This patch release turns run-level worktree discipline into a first-class Nexus
runtime contract. A fresh governed run now gets its own branch and linked
worktree automatically, execution stages stay bound to that workspace for the
life of the run, and CCB session discovery remains anchored at the repo root so
provider control does not fragment across worktrees.

## What Ships

- fresh `/discover` now allocates a dedicated `codex/run-*` branch and linked
  worktree under `.nexus-worktrees/` for each new run instead of reusing a
  stale implement worktree
- governed `/handoff`, `/build`, `/review`, `/qa`, and `/ship` now persist
  run-owned workspace and repo-root session-root provenance and execute against
  the bound workspace consistently
- `/closeout` now retires the run-owned worktree and the next fresh `/discover`
  safely removes clean retired worktrees or records them as retained when
  cleanup is unsafe

## Notes

- This is a patch release on top of `v1.0.25`.
- The control plane and execution plane are now split deliberately:
  - CCB session discovery and health checks stay at the repo-root session root
  - governed implementation stages execute inside the run-owned worktree
