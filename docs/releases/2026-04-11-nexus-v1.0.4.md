# Nexus v1.0.4

Date: 2026-04-11
Tag: `v1.0.4`
Status: `release_candidate`

## Summary

This release expands local execution topology support, makes Codex installation
usable as a conversational host flow, and absorbs more default stage content
into Nexus-owned runtime contracts.

## What Ships

- active local topology support for:
  - `claude_local + subagents`
  - `codex_local + subagents`
  - `codex_local + multi_session`
  - `gemini_local + subagents`
- stricter local topology gates so unsupported provider/topology combinations
  block at Nexus handoff instead of drifting into partial execution
- hardened local provider capability checks so non-zero CLI exits and missing
  subagent support surface as blocked Nexus state instead of false success
- Codex conversational install guidance in the public README plus clearer
  Codex post-install next-step messaging from `./setup --host codex`
- Nexus-owned absorbed default content for discover/frame/plan/build/review/qa/ship/closeout
  so live runtime outputs no longer depend on placeholder residue
- shared runtime prompt contracts and cleanup of Milestone 1 placeholder
  command/runtime remnants

## Notes

- `governed_ccb` remains the strict multi-provider governed path.
- `local_provider` is now materially broader, but legality and provenance still
  remain Nexus-owned and repo-visible.
- Gemini remains a supported provider, not a first-class installation host.
