# Nexus v1.0.21

Date: 2026-04-13
Tag: `v1.0.21`
Status: `release_candidate`

## Summary

This patch release closes the next governed retry gap exposed in real use.
Blocked `/handoff` attempts were leaving the run at `handoff` with no allowed
next stages, so the host surface treated the run as dead even though a retry
was the correct action.

## What Ships

- blocked or refused `/handoff` attempts now preserve `allowed_next_stages:
  ["handoff"]`, so the canonical next action is an immediate handoff retry
- a second `/handoff` on the same run now records canonical `via: "refresh"`
  provenance and can proceed without manual ledger edits
- `nexus-release-smoke` now bypasses stale raw GitHub manifest caching after
  publish, so maintainer release verification reads the current tag instead of
  occasionally reusing an older manifest

## Notes

- This is a patch release on top of `v1.0.20`.

  retry path open and still does not allow `/build` until route approval
  succeeds.
