# Nexus v1.0.51

Date: 2026-04-27
Tag: `v1.0.51`
Status: `release_candidate`

## Summary

This patch release keeps `/cso` security reports out of project worktrees by
default. Full CSO reports now go to local Nexus state, avoiding surprise
`.gitignore` prompts and reducing the risk that diagnostic security artifacts
are accidentally committed.

## What Ships

- `/cso` now writes full security reports under
  `~/.nexus/projects/${SLUG:-unknown}/security-reports/` by default.
- Project-local `.nexus/security-reports/` writes are now an explicit exception
  and require a `.gitignore` preflight before writing.
- The CSO trend-tracking instructions now read prior reports from local Nexus
  state.
- Regression coverage verifies that generated `/cso` guidance does not default
  to `mkdir -p .nexus/security-reports`.

## Verification

- `bun test test/gen-skill-docs.test.ts test/skill-e2e-cso.test.ts`
- `bun run build`
- `bun run skill:check`
- `git diff --check`

## Notes

- If a live `/cso` run on v1.0.50 prompts to add `.nexus/` to `.gitignore`, it
  is safe to choose `Yes` for that run. After upgrading to v1.0.51, `/cso`
  should no longer need that project-local write path unless explicitly asked.
