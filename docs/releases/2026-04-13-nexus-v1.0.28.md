# Nexus v1.0.28

Date: 2026-04-13
Tag: `v1.0.28`
Status: `release_candidate`

## Summary

This patch release fixes the remaining governed Phase-scoping hole in
fresh-run worktrees. Nexus now sends the current execution contract into
governed build and review prompts explicitly, so Codex evaluates the active
phase packet instead of falling back to an earlier acceptance slice that
happens to pass in the current checkout.

## What Ships

- governed `/build` now carries the handoff markdown plus the current
  execution-readiness packet and sprint contract as explicit predecessor
  artifacts
- governed `/review` now inherits the same execution contract artifacts, so
  full-acceptance review stays aligned with the same phase packet that drove
  build
- full-acceptance build prompts now explicitly tell the generator to implement
  the current execution packet instead of re-validating an earlier completed
  phase
- build no longer false-blocks when a generator correctly says it ignored stale
  `.planning/current/build/*` or `.planning/nexus/current-run.json` artifacts;
  the stale-artifact guard now detects actual reliance instead of raw string
  mentions
