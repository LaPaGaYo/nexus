# Nexus v1.0.47

Date: 2026-04-27
Tag: `v1.0.47`
Status: `release_candidate`

## Summary

This release tightens Nexus local-provider execution, completion advisor
interactions, support skill quality checks, and maintainer release diagnostics.
It also reduces generated design skill body length by moving dense design
knowledge into lazy-loaded references.

## What Ships

- local-provider startup checks now validate topology-specific requirements
  before offering a route that later lifecycle stages cannot run
- `/build` completion now keeps the visible post-stage chooser even when the
  build has no side-skill follow-up signal
- local-provider review and ship flows gain Nexus-native persona gates for
  multi-perspective checks without requiring CCB
- completion advisor recommendations now use checklist-backed rationale and
  stronger evidence signals
- support skill rubric and skill doctor checks cover anatomy, verification,
  overly broad triggers, canonical command conflicts, and generated host skill
  length regressions
- design-heavy skills now lazy-load compact reference files instead of carrying
  oversized active bodies
- `maintainer:check:report` provides a no-write local maintainer preview, while
  `--no-write` preserves strict gate exit codes for scripts
- local-provider routing and design voice prompts avoid Codex/Gemini assumptions
  when the active route is single-provider
