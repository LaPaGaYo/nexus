# Nexus v1.0.53

Date: 2026-04-27
Tag: `v1.0.53`
Status: `release_candidate`

## Summary

This patch release completes the Nexus skill source taxonomy migration and
separates PR landing from deployment. The source tree is now organized around
canonical lifecycle, support, safety, and alias skill categories while generated
host skills remain compatible with Claude, Codex, and factory installs.

## What Ships

- Source skills now live under `skills/canonical/`, `skills/support/`,
  `skills/safety/`, and `skills/aliases/`, with the root `/nexus` entrypoint
  kept at the repository root.
- Skill discovery, generation, validation, relinking, and e2e/static tests now
  resolve skills through the structured taxonomy instead of assuming flat
  source directories.
- `/land` is now a dedicated PR landing workflow for merge-without-deploy
  cases, while `/land-and-deploy` remains available for the full merge plus
  deploy flow.
- `/deploy` remains a separate post-land deployment workflow for projects with
  a configured deploy contract.
- Skill health checks now report the source taxonomy explicitly, including
  canonical/support/safety/alias counts and planned-move status.

## Notes

- Runtime sidecar asset directories such as `review/` and `cso/` intentionally
  remain in their existing locations because generated skills reference stable
  `$NEXUS_ROOT/...` paths.
- Existing generated host skill names keep the `nexus-*` prefix; the taxonomy
  migration affects source layout and generator internals, not user-facing
  command names.
