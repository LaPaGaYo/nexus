# Nexus v1.0.50

Date: 2026-04-27
Tag: `v1.0.50`
Status: `release_candidate`

## Summary

This patch release fixes the local-provider QA recovery path discovered after
v1.0.49: a blocked `/qa` can now retry `/qa` directly instead of being trapped
by lifecycle validation or misrouted into a build fix cycle.

## What Ships

- Blocked or refused QA states now allow a direct `/qa` retry.
- Completed QA failures with real defects still require the canonical `/build`
  fix cycle before QA can run again.
- The QA completion advisor now recommends `Retry /qa` for blocked/refused
  runtime failures instead of incorrectly recommending `/build`.
- Mirrored review audit artifacts under `.planning/audits/current` are treated
  as safe mirrored run artifacts during retired worktree cleanup.

## Verification

- `bun test`
- `bun run build`
- `bun run skill:check`
- `git diff --check`

## Notes

- Users blocked on v1.0.49 by `Illegal Nexus transition: qa -> qa` should
  upgrade to this release and retry `/qa`.
- Do not manually patch QA artifacts for this case; the runtime retry path now
  owns the recovery.
