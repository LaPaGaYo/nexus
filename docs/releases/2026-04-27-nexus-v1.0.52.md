# Nexus v1.0.52

Date: 2026-04-27
Tag: `v1.0.52`
Status: `release_candidate`

## Summary

This patch release fixes the `/ship` PR handoff order. Nexus now treats an
unpushed work branch as an explicit ship blocker instead of allowing `/closeout`
to continue after GitHub PR creation fails.

## What Ships

- `/ship` now checks local HEAD, upstream state, and existing PR head SHA before
  creating or reusing a pull request record.
- If the branch has no upstream, or local commits are ahead of upstream, `/ship`
  records `push_required`, stays not-ready, and surfaces the exact `git push`
  command as the required next action.
- `/closeout` remains blocked until `/ship` has a valid PR handoff, or the run
  explicitly does not require a PR.
- Generated `/ship` guidance now documents the durable order: commit stage
  artifacts, push branch, create or reuse PR, then closeout.

## Notes

- For an existing run that already reached `/ship` with an unpushed branch,
  push the branch first, then rerun `/ship`. Nexus should then create or reuse
  the PR handoff and allow `/closeout`.
