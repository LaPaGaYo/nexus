# Nexus v1.0.46

Date: 2026-04-26
Tag: `v1.0.46`
Status: `release_candidate`

## Summary

This patch release makes fresh Nexus run worktree branches provider-neutral.
New `/discover` runs now allocate `branch/run-*` branches instead of the legacy
`codex/run-*` namespace, so local Claude, local Codex, and governed CCB sessions
do not imply the wrong executor through branch names.

## What Ships

- fresh run worktree allocation now creates `branch/<run-id>` branches
- run-owned workspace detection recognizes both new `branch/run-*` and legacy
  `codex/run-*` branches
- discover/frame and workspace substrate tests now assert the provider-neutral
  branch namespace for new runs
- fresh-run worktree allocation docs now document the new branch namespace and
  legacy compatibility rule
