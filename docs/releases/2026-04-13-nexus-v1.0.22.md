# Nexus v1.0.22

Date: 2026-04-13
Tag: `v1.0.22`
Status: `release_candidate`

## Summary

This patch release closes a governed handoff verification-order bug exposed in
real CCB sessions. Nexus was pinging providers before giving `ccb-mounted
--autostart` a chance to bring the provider sessions up, so valid governed
handoff attempts could fail too early with "provider is not mounted".

## What Ships

- governed handoff now runs `ccb-mounted --autostart` before `ccb-ping <provider>`
- provider sessions that become available during `ccb-mounted --autostart` are
  now seen by the subsequent ping checks in the same handoff attempt
- route verification records now reflect the new verification order, with
  mounted discovery first and provider pings after
