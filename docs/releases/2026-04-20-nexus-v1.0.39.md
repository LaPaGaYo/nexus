# Nexus v1.0.39

Date: 2026-04-20
Tag: `v1.0.39`
Status: `release_candidate`

## Summary

This patch release turns stage-completion interaction into a runtime-owned
artifact contract instead of leaving it to prompt-only host behavior.

## What Ships

- canonical stages from `/frame` through `/closeout` now emit
  `.planning/current/<stage>/completion-advisor.json`
- advisor records now carry primary next actions, required-choice state,
  recommended side skills, default actions, and hidden alias/utility surfaces
- generated stage skills now consume advisor semantics instead of hardcoding
  next-step interaction logic in prose
- design-bearing runs now surface `/plan-design-review`, `/design-review`, and
  `/browse` from `design_impact` and the verification matrix through the same
  advisor layer
- manifest, docs, and product-surface tests now lock the advisor contract so
  interaction behavior stays governed and auditable

- This release is product-facing, not just internal cleanup. It makes
  stage-completion interaction durable and repo-visible.
- The interaction layer stays host-friendly but runtime-owned: canonical next
  steps now come from advisor artifacts, not only from generated prompt text.
