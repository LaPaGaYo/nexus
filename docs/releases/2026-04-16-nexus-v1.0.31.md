# Nexus v1.0.31

Date: 2026-04-16
Tag: `v1.0.31`
Status: `release_candidate`

## Summary

This patch release makes governed runs smoother across review and run
continuation. Nexus now records repo-visible CCB provider/request state,
bootstraps the next run directly from closeout artifacts, runs bounded dual
review in parallel with a tighter fast-auditor prompt, and recovers late review
results even when `ask` exits before surfacing a `CCB_REQ_ID`.

## What Ships

- governed CCB runtime now persists provider/session/request observability in
  `.planning/nexus/providers.json` and `.planning/nexus/dispatch/<request-id>.json`
- `/closeout` now emits `NEXT-RUN.md` and `next-run-bootstrap.json`, and fresh
  `/discover` seeds the next run from those artifacts with explicit
  `continuation_mode`
- governed `/review` now dispatches Codex and Gemini audits in parallel instead
  of running Gemini only after Codex completes
- bounded fix-cycle review prompts now force a fast bounded-auditor mode:
  inspect only the blocking-item files, avoid broad repo-search workflows, and
  finalize as soon as pass/fail evidence is sufficient
- governed CCB late-result recovery now also handles false starts where `ask`
  exits non-zero without a surfaced `CCB_REQ_ID` but provider output shows the
  review has already started
