# Nexus v1.0.25

Date: 2026-04-13
Tag: `v1.0.25`
Status: `release_candidate`

## Summary

This patch release fixes the remaining governed handoff/worktree bug exposed in
real CCB sessions. Nexus had already corrected `PWD`, but handoff route
verification was still running `ccb-mounted` and `ccb-ping` from the execution
worktree. When the project's `.ccb` sessions live at the main repo root, that
caused mounted providers to disappear during handoff even though the same
providers were healthy for the project.

## What Ships

- governed handoff now runs `ccb-mounted` and `ccb-ping` from the resolved CCB
  session root instead of the execution worktree
- build, review, and QA execution still run from the execution workspace, so
  worktree-based code changes remain isolated
- Nexus no longer needs hand-edited workspace artifacts to find mounted
  providers when the repo uses a linked worktree plus a root-level `.ccb`
  directory

- This is a patch release on top of `v1.0.24`.
- The fix deliberately separates two concerns:
  - CCB session discovery/control uses the repo session root
  - governed implementation still uses the selected execution workspace
