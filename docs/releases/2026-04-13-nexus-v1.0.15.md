# Nexus v1.0.15

Date: 2026-04-13
Tag: `v1.0.15`
Status: `release_candidate`

## Summary

This patch release fixes governed review freshness and retry behavior. Nexus now
forces a fresh provider session before governed review and QA dispatch, and a
failing or blocked review can rerun `/review` directly instead of forcing a
fake `/build` loop.

## What Ships

- governed `/review` and `/qa` now run `autonew` before dispatching to Codex or
  Gemini, so audit and QA work starts from a fresh provider session instead of
  stale session memory
- canonical transitions now allow `review -> review`, which gives governed
  retries a real lifecycle path when a review is stale, blocked, or otherwise
  needs to be rerun
- review state now preserves the original `previous_stage` across retries and
  records retry-safe `allowed_next_stages`:
  `['build', 'review']` after a failed gate and `['review']` after a blocked
  review dispatch
- closeout history now accepts repeated review entries, so a clean review retry
  does not force an unnecessary rebuild before `/closeout`
