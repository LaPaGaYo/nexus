# Nexus v1.0.14

Date: 2026-04-13
Tag: `v1.0.14`
Status: `release_candidate`

## Summary

This patch release fixes the governed stale-handoff recovery path. A fix-cycle
`/build` no longer consumes the state-machine transition when the handoff
contract is out of date, and Nexus now provides a canonical handoff refresh
path before retrying governed execution.

## What Ships

- `/build` now leaves the work unit on its prior stage when governed handoff
  verification fails before dispatch, instead of incorrectly advancing the run
  into a blocked `build` stage
- stale governed handoff packets now surface a single recovery step, rerun
  `/handoff`, rather than dead-ending the lifecycle with no legal transitions
- `/handoff` now supports canonical refresh from either an existing `handoff`
  stage or a failing `review` gate, while preserving the requested governed
  route instead of forcing manual artifact edits
- closeout history now accepts the explicit handoff-refresh form that can occur
  inside a governed fix cycle

- The user-visible recovery path after a stale governed handoff is now:
  `review -> handoff refresh -> build`, without hand-editing `.planning`
  artifacts.
