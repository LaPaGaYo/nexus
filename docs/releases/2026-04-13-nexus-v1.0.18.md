# Nexus v1.0.18

Date: 2026-04-13
Tag: `v1.0.18`
Status: `release_candidate`

## Summary

This patch release hardens governed runtime provenance after a full
discover-to-ship dogfood run. Nexus now records which workspace actually
executed the governed stages, keeps run-level route truth aligned with handoff,
rejects non-canonical ledgers before QA and ship, normalizes bounded fix-cycle
scope, and clarifies QA counts so successful verification evidence is no longer
misread as defect volume.

## What Ships

- governed execution now prefers an existing linked worktree workspace and
  persists workspace provenance across `/handoff`, `/build`, `/review`, `/qa`,
  and `/ship`
- run-level `route_check` now matches handoff `route_validation` as
  per-provider governed truth instead of retaining an older single-provider
  shape
- `/qa` and `/ship` now block when the run ledger is non-canonical, including
  malformed `command_history` or missing governed route-check evidence
- `command_history.via` now records canonical provenance such as
  `fix-cycle`, `retry`, and `refresh` instead of collapsing those transitions
  into `null`
- bounded fix-cycle `review_scope.blocking_items` now deduplicates equivalent
  Codex and Gemini findings before persisting downstream artifacts
- QA stage status now records `verification_count` and `defect_count` instead
  of the ambiguous `findings_count`

- Worktree support in this release is substrate-aware and provenance-complete,
  but it still prefers an existing linked worktree rather than auto-creating a
  new one.
