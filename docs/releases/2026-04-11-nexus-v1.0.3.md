# Nexus v1.0.3

Date: 2026-04-11
Tag: `v1.0.3`
Status: `release_candidate`

## Summary

This release turns execution topology into an explicit Nexus contract and makes
the Claude install path clearer when CCB is available.

## What Ships

- explicit `governed_ccb` and `local_provider` execution modes in Nexus ledger,
  stage status, routing, and reviewed provenance
- a formal local-provider runtime path for `/handoff`, `/build`, `/review`, and `/qa`
  when users do not want to run CCB
- blocked, repo-visible route decisions when CCB or the selected local provider CLI
  is unavailable instead of transport-layer crashes
- a slimmer Claude-facing instruction surface, with topic-specific guidance moved
  into `.claude/rules/`
- a managed global `~/.claude/CLAUDE.md` guidance helper for Nexus-first cross-project defaults
- an interactive `./setup` chooser that detects CCB and lets Claude users pick:
  current-session `local_provider` or tmux/CCB `governed_ccb`

- `subagents` and `multi_session` remain reserved local topology values for later activation.
