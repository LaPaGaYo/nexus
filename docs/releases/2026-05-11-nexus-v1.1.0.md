# Nexus v1.1.0

Date: 2026-05-11
Tag: `v1.1.0`
Status: `release_candidate`

## Summary

This release closes Phase 4 of Nexus and ships the first minor version since
v1.0.0. The runtime substrate is now organized by concern (18-bucket
`lib/nexus/` directory), `/nexus do "<intent>"` provides intent-driven routing
to canonical skills via the SkillRegistry, the 9 canonical lifecycle skills
carry numbered workflows + Iron Laws + AskUserQuestion gates under the
Track F darwin framework, and Track H wires Iron Law adoption telemetry into
the governed lifecycle.

## What Ships

### Structural reorganization

- `lib/nexus/` now organizes the runtime substrate by concern: 18 buckets
  including `cli/`, `commands/`, `adapters/`, `contracts/`, `governance/`,
  `runtime/`, `skill-registry/`, `intent-classifier/`, `completion-advisor/`,
  `observability/`, `release/`, `review/`, `io/`, and `skills/`. Internal
  imports updated mechanically; the user-facing `./bin/nexus <command>` shim
  is unchanged. (PR #137, PR #142)
- `vendor/upstream/` and `lib/nexus/absorption/` removed; Nexus-native stage
  content is the single source of truth, with no live runtime dependency on
  the historical upstream snapshot. (PR #126)
- `lib/nexus/index.ts` curated barrel introduced (19 value + 14 type exports)
  for cross-boundary callers; internal modules continue to import from
  concrete paths.
- Repo taxonomy refreshed (`docs/architecture/repo-taxonomy-v2.md`); scripts
  regrouped into `scripts/{build,skill,eval,repo,resolvers}/`. (PR #142)

### Routing & intent

- `/nexus do "<intent>"` dispatches free-form user intent against
  SkillRegistry manifest metadata with keyword match plus LLM tiebreaker;
  preserves governed lifecycle boundaries. (PR #122, #123, #128)
- SkillRegistry classifies skills into six namespaces (canonical, support,
  safety, aliases, root, external) for stage-aware advisor recommendations.
- Stage completion advisor
  (`.planning/current/<stage>/completion-advisor.json`) now surfaces
  `recommended_skills` and `recommended_external_skills` after every
  canonical stage. (PR #130)

### Skill workflow uplift (Track F)

- The 9 canonical lifecycle skills (`/discover`, `/frame`, `/plan`,
  `/handoff`, `/build`, `/review`, `/qa`, `/ship`, `/closeout`) now carry:
  - Iron Laws (non-negotiable constraints)
  - Numbered "How to run" workflows
  - AskUserQuestion gates at Iron Law branching points
  - Typical prompts with dry-run walkthrough
  - Completion Advisor footer
- Mean darwin-framework score across the 9 skills lifted from **73.1 → 86.0**,
  reaching control-group parity. See
  `docs/architecture/skill-strength-audit-v4-darwin-post-track-f.md`.

### Observability (Track H)

- Iron Law adoption telemetry: `/build`, `/qa`, `/ship`, and `/review` emit
  Iron Law adherence signals into the governed audit trail. (PR #124, PR #127)

### Bug fixes

- `/browse` sidecar self-watchdog and reap tooling for orphan headless
  Chrome processes. (closes #136)
- `local_provider` adapter tightens Codex/Gemini capability regex; blocks
  handoff if CLI lacks required command surface. (closes #103, PR #100)
- `nexus-relink` test suite no longer flakes on Bun's 5s default timeout
  (gives 30s to the 4-spawn tests). (closes #139)
- Stale upstream path removed from `nexus-relink`. (closes #132, #133)
- CI `unit-tests.yml` matrix runs each test file separately to sidestep
  Bun's auto-discovery exit-code aggregation bug. (closes #134, #135)
- `bin/nexus-refresh-follow-on-summary` path updated for the post-Phase-4
  reorg.
- Completion advisor shared helpers extracted; action IDs pinned. (closes #102)

### Documentation

- `docs/architecture/runtime-flow.md` — new mermaid invocation-flow diagram
  showing entrypoint → adapter → discovery → stage execution → artifact
  writes.
- `docs/architecture/context-diagram.md` refreshed for post-#137 paths.
- `docs/designs/SELF_LEARNING_V1_META.md` — new meta-spec mapping the
