# Nexus v1.0.33

Date: 2026-04-18
Tag: `v1.0.33`
Status: `release_candidate`

## Summary

This patch release finishes the remaining support-surface absorption work. The
mainline runtime now carries governed learnings, canonical deploy contracts,
verification matrices, retro continuity, archived attached evidence, and
auto-refreshed post-closeout follow-on summaries as first-class artifacts.

## What Ships

- fresh `/discover` now writes explicit session continuation advice and
  canonical follow-on context, while `/closeout` writes governed run learnings
  that `/learn` can read back from archived runs
- `/setup-deploy` now authors the canonical deploy contract under
  `.planning/deploy/`, `/ship` records deploy readiness from that contract, and
  `/land-and-deploy` reads ship-owned PR/deploy evidence and writes
  `.planning/current/ship/deploy-result.json`
- `/plan` now persists `.planning/current/plan/verification-matrix.json`, and
  `/handoff`, `/build`, `/review`, `/qa`, and `/ship` consume that matrix
  instead of relying on ad hoc verification intent
- retro history now archives under `.planning/archive/retros/`, and fresh
  discovery absorbs recent retros into repo-visible continuity artifacts
- attached evidence now behaves like governed provenance instead of stray
  sidecar output: performance verification, canary status, deploy results, and
  documentation sync records are archived with the run and folded into
  closeout-owned follow-on summaries
- post-closeout support workflows now refresh
  `.planning/current/closeout/FOLLOW-ON-SUMMARY.md` and
  `follow-on-summary.json`, while `nexus-ledger-doctor --fix-safe` can clean
  stale audits and stale attached evidence without touching command history
- governed review and QA status artifacts now surface human-readable CCB slow
  path diagnostics (`provider_slow`, `watchdog_recovery`,
  `orchestration_false_start`) instead of hiding them only in dispatch runtime
  state
