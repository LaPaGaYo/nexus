# Nexus v1.0.42

Date: 2026-04-25
Tag: `v1.0.42`
Status: `release_candidate`

## Summary

This patch release tightens Nexus as the workflow harness around provider
routing, design execution, and user-installed skill surfacing. Sessions now
start by choosing governed CCB or explicit local-provider mode, design
capabilities are absorbed into Nexus-native skills without exposing a second
product surface, and external installed skills can appear as supplemental
advisor recommendations without overriding Nexus lifecycle commands.

## What Ships

- `/nexus` now detects whether governed CCB is available and falls back to an
  explicit local-provider path with recorded provider/topology provenance
- governed review transport now has stronger watchdog/receipt gating so stale
  provider replies do not overwrite the current audit truth
- the absorbed design deliverable engine now lives behind Nexus-native design
  skills, with stronger design lifecycle integration and no separate
  `huashu-design` product surface
- completion advisors can scan installed `SKILL.md` files and surface matching
  external skills through `recommended_external_skills`
- Nexus canonical commands and Nexus support skills remain higher priority than
  any installed external skill with a colliding name or overlapping purpose

## Notes

- External installed skills are supplemental context only. They do not become
  lifecycle stages, support-skill owners, or governed artifact truth.
- Design work should still route through `/design-consultation`,
  `/design-shotgun`, `/plan-design-review`, `/design-html`, `/design-review`,
  and `/browse` as Nexus-owned surfaces.
