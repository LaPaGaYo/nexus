# Nexus v1.0.32

Date: 2026-04-16
Tag: `v1.0.32`
Status: `release_candidate`

## Summary

This patch release packages the remaining unreleased governed runtime work into
one stable update. Nexus now treats design-bearing work as a first-class
contract in the main lifecycle, makes governed review faster and more reliable
under CCB failure modes, exposes explicit continuation control for fresh runs,
automates the maintainer publish path, records PR handoff at `/ship`, and adds
better diagnosis for noncanonical closeout histories.

## What Ships

- `design_impact` now flows through the canonical lifecycle:
  `material` UI work must carry `.planning/current/plan/design-contract.md`,
  governed build/review consume that contract, QA writes
  `.planning/current/qa/design-verification.md`, and `/ship` blocks
  design-bearing runs that never recorded visual verification
- governed CCB review now has a stricter request lifecycle:
  repo-visible dispatch/provider state, bounded review watchdogs, finalize
  nudges, slow-lane `pend` recovery, and silent non-zero `ask` exits no longer
  block review too early
- fresh `/discover` now accepts explicit continuation intent through
  `--continuation-mode task|phase|project_reset` (with
  `NEXUS_CONTINUATION_MODE` as the automation fallback) instead of forcing
  every new run through an inferred default
- release/maintainer publication now has a canonical atomic path via
  `./bin/nexus-release-publish`, including skill-doc freshness checks, ordered
  push/tag/release/smoke execution, and post-release maintainer refresh
- `/ship` now owns GitHub PR handoff through
  `.planning/current/ship/pull-request.json`; `/land-and-deploy` consumes that
  metadata instead of assuming PR creation happens at `/closeout`
- Nexus now reports noncanonical run state explicitly:
  `/closeout` explains the exact illegal history segment, and
  `./bin/nexus-ledger-doctor` reports `history_mismatch` and
  `stale_current_audits` without mutating the repository

## Notes

- This is the next patch candidate on top of `v1.0.31`.
- The canonical lifecycle remains
  `/discover -> /frame -> /plan -> /handoff -> /build -> /review -> /qa -> /ship -> /closeout`;
  this release tightens the contracts, support workflows, and maintainer path
  around that lifecycle rather than adding a new stage.
