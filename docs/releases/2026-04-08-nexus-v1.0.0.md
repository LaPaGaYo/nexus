# Nexus v1.0.0

Date: 2026-04-08
Tag: `v1.0.0`
Status: `released`

## Summary

This is the first full Nexus release.

Nexus now presents as one unified system:

- one command surface
- one governed lifecycle
- one repo-visible state model
- one audit and closeout model
- one runtime ownership layer

The old source systems no longer define the product surface:

- Gstack host architecture was absorbed into a Nexus-first shell
- PM Skills methods were absorbed into Nexus discovery and framing
- GSD methods were absorbed into Nexus planning and closeout
- Superpowers discipline was absorbed into Nexus build, review, QA, and ship
- CCB remains connected underneath as dispatch and transport infrastructure only

## What Ships

- Canonical lifecycle commands:
  - `/discover`
  - `/frame`
  - `/plan`
  - `/handoff`
  - `/build`
  - `/review`
  - `/qa`
  - `/ship`
  - `/closeout`
- Nexus-owned runtime and contracts in `lib/nexus/`
- Repo-visible governed artifacts in `.planning/`
- Nexus-first helper surface:
  - `nexus-config`
  - `nexus-relink`
  - `nexus-uninstall`
  - `nexus-update-check`
- Real CCB dispatch integration for governed cross-model transport

## Operating Model

The intended usage path is now:

1. Start `tmux`
2. Start `ccb claude codex gemini`
3. Enter Claude
4. Run work through Nexus

Operationally:

- the human interacts with Claude
- Claude invokes Nexus
- Nexus owns commands, lifecycle, artifacts, and governance
- Nexus dispatches external model work through CCB when required
