# Nexus v1.0.30

Date: 2026-04-15
Tag: `v1.0.30`
Status: `release_candidate`

## Summary

This patch release hardens governed review/build recovery after broken or late
dual-audit runs. Nexus now cleans stale current audit artifacts when a review
blocks, recovers late CCB audit payloads via `pend`, and refuses fix-cycle
`/build` unless the predecessor review is a complete canonical failing review.

## What Ships

- governed CCB review dispatch now polls `pend` when `ask` exits non-zero after
  emitting a `CCB_REQ_ID`, so late Gemini or Codex audit payloads can still be
  recovered and normalized into the current review
- blocked or refused `/review` no longer leaves stale
  `.planning/audits/current/*` files in place as current truth; Nexus clears the
  current audit set and removes the corresponding run-ledger artifact pointers
- fix-cycle `/build` now requires a complete canonical failing review
  (`completed` + `audit_recorded` + complete audit set + consistent provenance),
  instead of admitting any review status that merely says `gate_decision: fail`
